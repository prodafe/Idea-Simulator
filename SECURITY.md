# 安全政策

## 报告漏洞

如果发现安全漏洞，请通过 GitHub Issues 的 "Security Advisory" 功能私下报告，或发送邮件至项目维护者。请勿公开披露。

## 安全特性

- **API密钥**: 仅用于当次推演，不存储、不记录
- **限流**: 每分钟60次请求，防止滥用
- **鉴权**: API端点需要 X-API-Key 头
- **无数据收集**: 不收集用户输入数据，不追踪用户行为

## 已知限制

- 本项目为本地部署，不提供托管服务
- API密钥由用户自行管理
- 数据文件为公开数据，无敏感信息

## 依赖安全

定期检查 `requirements.txt` 和 `pyproject.toml` 中的依赖版本。建议使用 `pip-audit` 扫描已知漏洞。
